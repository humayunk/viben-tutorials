What is OpenClaw?

OpenClaw (formerly Clawdbot / Moltbot) is a self-hosted AI agent that connects to your messaging apps — WhatsApp, Telegram, Discord — and can execute real tasks on your computer: manage your calendar, triage emails, check Git repos, send messages, and more.

The key concept is the "Gateway" — a long-running background process on your machine that maintains persistent connections to messaging platforms, routes incoming messages to an AI agent, and lets the agent take action.

How is it different from Claude Code? Fully self-hosted (you own everything), more integrations (WhatsApp, Telegram, Discord, email), more configurable, and runs 24/7 as a background daemon.

Before You Start

You'll need:

  • Node.js v22+ installed
  • Basic comfort with terminal (or willingness to copy-paste commands)
  • Some exposure to AI chat (you've used ChatGPT, Claude, etc.)
  • An API key from Anthropic, OpenAI, or Google
Security Decision OpenClaw gets full access to your computer — terminal, files, everything. For learning, running locally is fine. For production, consider a VPS or Docker sandboxing (Step 8). Using a more powerful model (Claude Opus) is actually more secure — larger models resist prompt injection better.
1
Install OpenClaw
# Install globally via npm
npm install -g openclaw@latest

# Run the onboarding wizard + install as background service
openclaw onboard --install-daemon

The --install-daemon flag installs the gateway as a background service (launchd on Mac, systemd on Linux) so it starts automatically on boot. No terminal window needed.

3
Meet Your Agent
# Launch the terminal interface
openclaw

On first boot, the agent introduces itself and asks you to name it. Give it a name, tell it about yourself. The more context you provide, the better it works.

What happens behind the scenes: The agent creates markdown files in ~/.openclaw that define everything about it:

~/.openclaw/
  agent.md ← Core instructions (most critical file)
  identity.md ← Name, personality, role
  memory.md ← What it remembers about you
  user.md ← Info about you (name, timezone)
  heartbeat.md ← Periodic tasks (runs hourly)
  soul.md ← Agent identity definition
  tools.md ← Scratchpad for tool interactions
  skills/ ← Installed skills
  config/ ← Auth profiles, settings
Key insight Your agent is defined entirely by markdown files. You can edit them directly, back them up to Git, and sync across devices. Switch machines? Just pull the repo.
4
Run Security Checks

Before doing anything else:

# Deep security audit
openclaw security audit deep

# Health check for the gateway
openclaw doctor

# Auto-fix any issues
openclaw doctor fix

The security audit checks for files with too-broad permissions (e.g., executable files that shouldn't be). The doctor checks the gateway process health. Fix everything it flags.

5
Connect Messaging Channels

WhatsApp

# Enable WhatsApp plugin
openclaw plugins enable whatsapp

# Restart gateway
openclaw gateway restart

# Login (shows QR code to scan)
openclaw channels login

Scan the QR code with your phone's WhatsApp. Then add your phone number to the config.

Never put your bot in a group chat Anyone in the group could send prompts that execute on your machine. Keep it 1-on-1 only. Ideally, use a dedicated second phone number for the bot.

Discord

Ask your agent in the TUI: "Help me set up a Discord bot." It walks you through creating the application, getting the token, and configuring everything. Same security rule: use a private server where only you and the bot are members.

Email

OpenClaw can send emails via a custom skill using Gmail SMTP. If you have 2FA, you'll need a Google App Password.

6
Add Skills

Skills are markdown files with YAML front matter that teach the agent how to use specific tools. Think of them as instruction manuals for the AI.

SkillWhat it does
ObsidianManage your vault, query notes
Apple NotesRead/write via memo CLI
Bird (Twitter)Tweet, thread, retweet
HimalayaRead, write, reply to emails
GoogleGmail, Calendar, Docs
1PasswordCredential management
Nano BananaImage generation
 
# Browse skills at clawhub.com, install like pip:
clawhub install [skill-name]

You can also ask your agent to write custom skills. Example: "Write an email skill using Python that sends via Gmail SMTP." It creates the skill.md and script, stores it in ~/.openclaw/skills/, and you can invoke it with /skill [name].

Treat third-party skills as untrusted code Always read the skill.md before enabling. Check for anything suspicious or overly broad.
7
Multi-Agent Setup

Run multiple agents with different personas, permissions, and workspaces.

# Add a new agent called "work"
openclaw agents add work

# List all agents
openclaw agents list

# Switch agents in the TUI
/agents  # arrow key to select

Use case: A "work" agent with Slack and email integrations and access to work repos. A "personal" agent with calendar, smart home, and social media. Switch between them with /agents. Each gets its own workspace, memory, auth profile, sandbox settings, and tool permissions.

8
Docker Sandboxing

The middle ground between "full access to my Mac" and "run on a VPS."

# Make sure Docker is running, then:
scripts/sandbox-setup.sh

# Or ask your agent:
# "Make the work agent sandboxed on agent scope"
ModeWhat it does
noneNo sandboxing (default)
non-mainSandbox all agents except default
allSandbox everything
ScopeContainer lifecycle
sessionNew container per session (most isolated, most overhead)
agentOne container per agent (good balance)
sharedAll sandboxed agents share one container

Verify it works by switching to the sandboxed agent and asking "Can you access my desktop files?" — it should say "Nope, I'm sandboxed."

The elevated tool policy bypasses the sandbox Never grant elevated access to agents handling untrusted input.

What to Try First

  • Install and onboard — get the TUI running
  • Have a conversation — name your agent, tell it about yourself
  • Run security audit — fix anything it flags
  • Connect WhatsApp — send yourself a message through the bot
  • Try a skill — install Obsidian or email skill
  • Set up Docker sandbox — if you plan to do anything risky
